  title: 81 Pass Checker API
  description: |
    ## Visión General
    **81 Pass Checker** es una plataforma de **Verifiable Credentials (VC)** y motor de elegibilidad en tiempo real que permite a organizaciones emisoras validar credenciales digitales W3C y evaluar la elegibilidad de usuarios mediante lógica de negocio declarativa.

    ### Arquitectura Core
    - **Motor de Decisión**: Evaluación de condiciones complejas (certificados, cursos, atributos biométricos, temporalidad) contra credenciales verificables
    - **Gestión de Ciclo de Vida**: Emisión, validación, revocación y auditoría de credenciales con trazabilidad completa
    - **Autorización Granular**: Control de acceso basado en roles (RBAC) para emisores verificados

    ### Casos de Uso Principales
    - Verificación de identidad descentralizada (DID)
    - Elegibilidad para beneficios gubernamentales o corporativos
    - Acceso condicional a servicios basado en atributos verificables
    - Cumplimiento normativo KYC/AML con pruebas de vida (liveness detection)

    ### Seguridad & Compliance
    - Autenticación mediante API Keys con rotación automática
    - Encriptación en tránsito (TLS 1.3) y en reposo (AES-256)
    - Trazabilidad completa mediante logs inmutables (append-only)
    - Cumplimiento W3C Verifiable Credentials Data Model 2.0

    ### Rate Limits 
    - **Tier Estándar**: 100 req/min por emisor
    - **Tier Enterprise**: 1000 req/min con burst allowance
    - **Retry-After**: Incluido en headers de respuesta 429

  /engine/checker:
    post:
      summary: Ejecutar Motor de Elegibilidad (Eligibility Engine)
      description: |
        Evalúa un conjunto de condiciones complejas contra las credenciales verificables 
        de un usuario específico para determinar elegibilidad en tiempo real.

        ### Lógica de Evaluación
        El motor procesa condiciones como predicados lógicos soportando:
        - **Atributos demográficos**: Edad, nacionalidad, residencia
        - **Credenciales temporales**: Fechas de emisión/expiración, validez de pruebas de vida
        - **Credenciales de Carrera**: Certificados, cursos etc.
        - **Credenciales de Identidad**: Email, Telefono etc.
        - **Composición lógica**: AND/OR/NOT anidados

        ### Algoritmo de Matching
        1. **Resolución de DID**: Identificación descentralizada del usuario
        2. **Recuperación de VC**: Query a almacenamiento cifrado de credenciales
        3. **Verificación criptográfica**: Validación de firmas de emisor
        4. **Evaluación de reglas**: Aplicación de motor de decisión
        5. **Generación de prueba**: Creación de Verifiable Presentation (VP) opcional

        ### Ejemplo de Condición Compleja
        ```json
        {
          "operator": "AND",
          "conditions": [
            {"field": "credential.type", "operator": "eq", "value": "NationalID"},
            {"field": "claims.age", "operator": "gte", "value": 18},
            {"field": "metadata.liveness_check", "operator": "lt", "value": "24h"}
          ]
        }
        ```
      tags:
        - Motor de Elegibilidad
      operationId: executeEligibilityCheck
      requestBody:
        required: true
        description: Criterios de evaluación y contexto del usuario
        content:
          application/json:
            schema:
              $ref: "#/components/schemas/EligibilityRequest"
            examples:
              simpleCheck:
                summary: Verificación simple de edad
                value:
                  user_id: "did:81pass:user_9xYzA1"
                  conditions:
                    - field: "claims.age"
                      operator: "gte"
                      value: 18
                  context:
                    purpose: "Access to restricted content"
                    requester_id: "org_2vPqN5LwX8kT"
              complexCheck:
                summary: Verificación compuesta con geolocalización
                value:
                  user_id: "did:81pass:user_9xYzA1"
                  conditions:
                    operator: "AND"
                    rules:
                      - field: "claims.city_of_birth"
                        operator: "eq"
                        value: "Cali"
                      - field: "metadata.liveness_timestamp"
                        operator: "gt"
                        value: "2024-01-01T00:00:00Z"
                      - field: "credentials.status"
                        operator: "eq"
                        value: "active"
                  context:
                    purpose: "Government benefit eligibility"
                    geofence: "CO-VALLE-CALI"
      responses:
        "200":
          description: Evaluación completada exitosamente
          content:
            application/json:
              schema:
                $ref: "#/components/schemas/EligibilityResponse"
              examples:
                matchResult:
                  summary: Usuario elegible
                  value:
                    result: "MATCH"
                    confidence_score: 0.98
                    matched_credentials:
                      ["cred_national_id_001", "cred_liveness_002"]
                    verification_proof: "vp_token_eyJhbGciOiJFUzI1NiIs..."
                    evaluated_at: "2024-01-15T10:30:00Z"
                noMatchResult:
                  summary: Usuario no elegible
                  value:
                    result: "NO_MATCH"
                    confidence_score: 0.0
                    missing_requirements:
                      - "Liveness check expired (>24h)"
                      - "Missing residency credential"
                    suggestions:
                      - "Re-validate liveness detection"
                      - "Request address verification"
                    evaluated_at: "2024-01-15T10:30:00Z"
        "400":
          $ref: "#/components/responses/BadRequestError"

          content:

  /credentials/validate:
    post:
      summary: Validar Credencial Específica (Cryptographic Verification)
      description: |
        Realiza verificación criptográfica completa de una credencial presentada,

        ### Proceso de Validación
        1. **Parseo de JWT-VC**: Decodificación del credential token
        2. **Verificación de firma**: Validación contra DID Document del emisor
        3. **Chequeo de revocación**: Consulta a VC Revocation List 2020
        4. **Validación de schema**: Verificación contra JSON Schema del tipo de credencial
        5. **Verificación de expiración**: Chequeo de fechas `validFrom` y `validUntil`

        ### Tipos Soportados
        - W3C Verifiable Credentials (JWT-VC y LD-Proofs)
        - ISO mDL (Mobile Driver License) - *Próximamente*
      tags:
        - Gestión de Credenciales
      operationId: validateCredential
      requestBody:
        required: true
        content:
          application/json:
            schema:
              type: object
              required: ["credential_jwt", "challenge"]
              properties:
                credential_jwt:
                  type: string
                  description: Credencial en formato JWT-VC codificado
                  example: "eyJhbGciOiJFUzI1NiIsInR5cCI6IkpXVCJ9..."
                challenge:
                  type: string
                  description: Nonce criptográfico para prevenir replay attacks
                  example: "nonce_8f7d9a2b3c4e5f6g"
                verify_revocation:
                  type: boolean
                  description: Forzar verificación contra lista de revocación
                  default: true
      responses:

  /credentials/revoke:
    patch:
      summary: Revocar Credencial (Revocation)
      description: |
        Invalida una credencial previamente emitida, agregándola a la Lista de Revocación 
        Verificable (VC-RL) y notificando a los verificadores suscriptos.

        ### Consideraciones Importantes
        - **Irreversibilidad**: La revocación es permanente (no se puede deshacer)
        - **Propagación**: La revocación puede tardar hasta 5 minutos en propagarse globalmente
        - **Notificación**: Se envía webhook a verificadores que hayan consultado la credencial recientemente
        - **Auditoría**: Requiere justificación detallada y autorización de segundo factor

        ### Causales de Revocación Válidas
        - `COMPROMISED`: Clave privada del titular comprometida
        - `EXPIRED`: Renovación o reemplazo por nueva versión
        - `FRAUD`: Detección de emisión fraudulenta
        - `REQUESTED`: Solicitud voluntaria del titular
        - `ADMINISTRATIVE`: Error administrativo del emisor
      tags:
        - Gestión de Credenciales
      operationId: revokeCredential
      requestBody:
        required: true
        content:
          application/json:
            schema:
              type: object
              required: ["credential_id", "reason", "justification"]
              properties:
                credential_id:
                  type: string
                  description: ID único de la credencial a revocar
                  example: "cred_national_id_001"
                reason:
                  type: string
                  enum:
                    [
                      "COMPROMISED",
                      "EXPIRED",
                      "FRAUD",
                      "REQUESTED",
                      "ADMINISTRATIVE",
                    ]
                  description: Causal de revocación
                justification:
                  type: string
                  minLength: 20
                  description: Explicación detallada del motivo (mínimo 20 caracteres)
                  example: "Usuario reportó pérdida de dispositivo móvil con credencial almacenada"
                notify_holder:
                  type: boolean
                  default: true
                  description: Notificar al titular de la credencial vía email/SMS
      responses:
        "200":
          description: Credencial revocada exitosamente
          content:
            application/json:
              schema:
                type: object
                properties:
                  credential_id:
                    type: string
                  status:
                    type: string
                    enum: ["revoked"]
                  revoked_at:
                    type: string
                    format: date-time
                  revocation_list_index:
                    type: integer
                    description: Posición en la lista de revocación merkle tree
                  affected_verifiers:
                    type: integer
                    description: Número de verificadores notificados
        "409":
          description: Credencial ya revocada previamente
        "422":
          description: Justificación insuficiente o causal inválida
